# Implementation Plan: Log Viewer

**Branch**: `008-log-viewer` | **Date**: 2026-02-11 | **Spec**: `specs/008-log-viewer/spec.md`
**Input**: Feature specification from `/specs/008-log-viewer/spec.md`

## Summary

Implement a Log Viewer dialog that lets registered users search, browse, filter, and export their persisted chat history (channels and PMs). The feature reads from existing `messages` and `private_messages` tables using new composable Ecto queries with offset-based pagination. No new migrations are required — all new data structures (LogFilter, DisplayPreferences, LogPage) are in-memory only. Export uses the existing `push_event` + JS hook pattern to trigger browser-side file downloads. Guest users see only their current session's in-memory messages.

## Technical Context

**Language/Version**: Elixir 1.17+ / OTP 27+
**Primary Dependencies**: Phoenix 1.8+, Phoenix LiveView 1.0+, Ecto 3.x, retro design system
**Storage**: PostgreSQL 16+ (existing `messages` and `private_messages` tables, read-only — no new migrations)
**Testing**: ExUnit with async: true, Floki for LiveView tests, TDD per Principle IV
**Target Platform**: Web (Phoenix LiveView)
**Project Type**: Umbrella (retro_hex_chat domain + retro_hex_chat_web)
**Performance Goals**: First page of 50 results in <2 seconds for queries spanning up to 30 days
**Constraints**: Offset-based pagination acceptable for filtered result sets; ILIKE sufficient for text search at current scale
**Scale/Scope**: Single dialog with ~6 domain modules, ~1 component, ~15 LiveView event handlers, 1 JS hook
