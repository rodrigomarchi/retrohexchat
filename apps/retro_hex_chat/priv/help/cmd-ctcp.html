/ctcp

Query information about another user's client or measure your connection latency to them using the Client-to-Client Protocol.

Syntax

/ctcp <nickname> <type>

Parameters

nickname — The user to query.
type — One of: ping, version, time, finger.

How It Works

CTCP sends a private request to the target user's client. The response depends on the type: ping measures round-trip latency in milliseconds, version shows the target's client name and version, time returns the server's current UTC time, and finger returns the user's profile text or idle time. Responses appear only to you — CTCP exchanges are invisible to other users and do not create PM windows.

Rules & Limits

  • Rate limited to 3 requests per target per 30 seconds
  • Requests time out after 10 seconds if the target doesn't respond
  • You can CTCP yourself for instant responses

Examples

/ctcp Alice ping
/ctcp Bob version
/ctcp Alice time
/ctcp Bob finger
